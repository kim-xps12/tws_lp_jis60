### 1.1 このキーボードの特徴

`tws_lp_jis60` は **JIS配列・ロープロファイル・完全無線** の60%左右分割キーボードです。元ネタは [sphh jp v2 (a bit keys)](https://abitkeys.com/sphhjp/build_v2.html) で、HHKB-likeなJIS配列を分割化したコンセプトを継承しつつ、無線化とロープロファイル化を加えています。

- **JIS 60% 配列**: ISOエンター、矢印キー、`全/半`・`無変換`・`変換`・`かな` などのJIS変換キーを備え、慣れた配列のまま分割キーボードに移行できます。
- **完全無線**: PC接続にBLE、左右間の通信も無線。ケーブル一切なしでデスクをすっきり保てます。
- **ロープロファイル**: Kailh Choc v2 互換スイッチを採用し、手首への負担が少ない打鍵姿勢を実現。
- **ホットスワップ対応**: スイッチを後から差し替え可能。はんだ付けはせず気軽にカスタムできます。
- **PCBサンドイッチ構造**: 上下プレートも基板で構成し、剛性・安定感のある打鍵感に。

## 4. はんだ付け

> 本キットの実装順は次のとおりです。
>
> 1. ダイオード（表面実装、左43個／右53個）
> 2. キースイッチソケット（裏面実装、左30個／右39個）
> 3. XIAO の取り付け（表面実装）
> 4. リポバッテリーの配線（XIAO 側）
> 4. リポバッテリー用コネクタ
>
> 左右で部品点数が異なるので、進めながら数を確認してください。

### 4.1 表裏の確認

本キットは **表裏で実装する部品が分かれます**。

- **表面（キースイッチが刺さる側）**: XIAO
- **裏面**: Kailh Choc V2 ホットスワップソケット，SMDダイオード

基板の裏表および左右をよく確認してから作業を始めてください。

> **TBD**: 左右・表裏の判別方法（シルクの具体的な表記）— 設計者からの情報待ち

### 4.2 ダイオード（1N4148W）

#### 概要

- **部品**: 1N4148W チップダイオード（SOD-323 等のSMDパッケージ）
- **実装方法**: 基板パッドに直接はんだ付けする表面実装（SMD）
- **数量**: 左 43個 / 右 53個
- **実装面**: 基板の表面

#### 極性

チップダイオードには線（バンド）が入っています。この線側が **カソード（K）** です。

基板シルクの白いライン（または三角形の先端）がカソード側を示します。**必ず線とシルクの向きを揃えて** ください。逆向きに付けると該当キーが反応しません。

> 70個近くを全部同じ向きに揃える必要があります。先に基板を見て、すべてのダイオードの向きが揃っているか／一部だけ反対向きか を確認してから取り付けに入ると安心です。

#### 取り付け手順

1. パッドの **片側だけ予備はんだ** をします。
2. ピンセットでダイオードを保持し、予備はんだをこて先で溶かしながら部品を着地させます。極性を確認。
3. 部品が水平に固定されたことを確認して、**反対側のパッド** をはんだ付けします。
4. 最初の予備はんだ側に戻り、必要ならはんだを少量足して整えます。

これを 左43回 / 右53回 繰り返します。途中で疲れたら休憩を挟んでください。集中力が切れると向きを間違えやすくなります。

#### よくある失敗

- **極性逆**: 該当キーが反応しないので、書き込み後の動作確認で気づきます。フラックスを塗って片側のはんだを溶かし、ピンセットで外して付け直してください。
- **はんだ付け不良（イモはんだ）**: 表面が艶のないザラついた仕上がりだと接触不良の原因に。フラックスを足して再加熱で改善。
- **ブリッジ**: 隣接するパッド間がはんだで繋がってしまう状態。はんだを盛り過ぎないよう注意。発生した場合はフラックスを塗って吸取線で除去。

### 4.3 Kailh Choc V2 ホットスワップソケット

#### 概要

- **部品**: Kailh Choc V2 ホットスワップソケット
- **実装面**: 基板の **裏面**
- **数量**: 左 30個 / 右 39個

> ソケットは基板の **裏面** に取り付けます。表面のキースイッチホールから足が出ないため、表裏を間違えるとスイッチが刺さらなくなります。

#### 取り付け手順

ソケットの2つの金属パッドを基板裏面のパッドに合わせてはんだ付けします。

1. ソケットを基板の裏面に正しい向きで配置します。基板側の白いシルク枠とソケット形状を合わせます。
2. 片方のパッドに **片側予備はんだ** をしておきます。
3. ピンセットでソケットを押さえ、予備はんだを溶かして片側を固定。
4. 位置と密着を確認したら、**反対側のパッド** をはんだ付け。
5. 最初の側に戻ってフィレットを整える。

> ソケットの金属パッドはやや大きいので、はんだごての熱が逃げやすいです。温度を少し高めに設定するか、こて先をパッドにしっかり当ててください。

#### 装着確認

全ソケット取り付け後、手元のキースイッチを **1個だけ** 取って差し込んでみてください。

- カチッとした感触で奥まで入る → OK
- グラグラする / 奥まで入らない → ソケットの浮き、向き間違い、はんだ不良の可能性

このチェックを **ファーム書き込み前** にやっておくと、後工程でのトラブルを切り分けやすくなります。

### 4.4 XIAO の取り付け

#### 概要

XIAO nRF52840 は本体基板に **直接はんだ付け** で取り付けます。

> 直接はんだ付けのため、取り付け後の取り外しは困難です。はんだ付け前に XIAO 本体の動作確認（USBで認識されるか）を済ませておくことをおすすめします。

#### 取り付け手順

1. 本体基板の表面、XIAO用フットプリント位置に XIAO を載せます。
2. XIAO が基板と平行になっていることを確認し、**対角の1ピンずつ仮はんだ** して位置を固定します。傾いていたらはんだを溶かして修正。
3. 残りのピンをはんだ付けします。1ピンずつ間隔を空けながら付けると熱が一点に溜まらず作業しやすいです。


#### 安全上の注意（再掲）

リポバッテリーは取り扱いを誤ると発火・爆発の危険があります。次の点を厳守してください。

- **+/− を絶対に間違えない**（逆接続でショート→発火の危険）
- 配線中はバッテリーを **絶縁テープで覆う** などしてショートを防止
- 膨らみ・変色・異臭を感じたら即座に使用中止
- はんだごてをバッテリー本体に近づけない

#### 取り付け手順

1. 基板裏面の、XIAO の `BAT+` / `BAT-` に対応するスルーホールの位置を確認します。
2. バッテリーのリードを必要な長さにカットし、先端の被覆を剥がして予備はんだします。
3. バッテリーのリードを基板裏面のスルーホールに通し、XIAO 背面の BAT パッドとつなぐようにはんだ付けします（**極性を必ず確認**）。
4. 取り付け後、テスターで `BAT+` と `BAT-` 間にバッテリーの電圧（3.7V前後）が出ているか確認。
5. バッテリー本体を本体基板の所定位置に固定します。

> **TBD**: バッテリーの固定方法（両面テープ／ケース内の専用受け／その他）

### 4.6 はんだ付け完了後のチェック

ファームウェアを書き込む前に、次の項目を目視+テスターで確認しておきます。

- [ ] ダイオードの向きが全て揃っている
- [ ] ホットスワップソケットの浮き・はんだ不良がない
- [ ] XIAO が基板と平行に取り付けられている
- [ ] XIAO の全ピンがはんだ付けされている
- [ ] バッテリー極性が正しい
- [ ] BAT+ と BAT- 間で短絡していない（テスターで確認）

問題なければ「5. 組み立て」に進みます。

## 5. 組み立て

はんだ付けが終わったら、上下プレートで基板を挟む **サンドイッチ構造** に組み立てます。

### 5.1 構成

本キーボードは次の層構造です。

```
┌──────────────────┐   トッププレート (PCB)
│                  │
└─┬──────────────┬─┘   ← スペーサー / ネジ
  │              │
  │  メイン基板  │
  │              │
┌─┴──────────────┴─┐   ← スペーサー / ネジ
│                  │
└──────────────────┘   ボトムプレート (PCB)
```

- **トッププレート**: スイッチ取付穴あり。はんだ付け不要の純粋なPCBプレート
- **メイン基板**: スイッチソケット・ダイオード・XIAOが載る本体
- **ボトムプレート**: 底面の保護とXIAO/バッテリー収納部のカバー

> 中間プレートはありません。トッププレート / メイン基板 / ボトムプレートの **3層構成** です。

![サンドイッチ構造分解図](images/05_sandwich.jpg)

> ネジ規格は **M2** です。左右各 **8箇所** のネジ穴があります。スペーサーの長さは **TBD**（設計者に確認中）。

### 5.3 組立手順

ネジ穴の位置に注意しながら、次の順で組み立てます。

1. **ボトムプレートを下に置く**
2. **下段スペーサーを各ネジ穴に立てる**
   - 必要に応じて下からネジを差し込み、スペーサーをねじ込んで固定
3. **メイン基板を載せる**
   - ソケット・XIAO・バッテリーが下を向く（=ボトム側に収まる）向き
   - ネジ穴が一致するか確認
4. **上段スペーサーを取り付ける**
5. **キースイッチを2〜3個、対角の位置に仮挿入**
   - メイン基板のソケットとトッププレートの穴が揃っているか確認するため
6. **トッププレートを載せる**
   - 仮挿入したスイッチがトッププレートの穴に通ることを確認
7. **上からネジを締めて固定**
   - 4隅から対角線順に締め、徐々に各ネジを均等に増し締め
   - 過度に締めると基板やネジ山を傷めるので **手締めで十分**

## 6. ファームウェア（ZMK）

### 6.1 概要

本キットのファームウェアは [ZMK Firmware](https://zmk.dev/docs) をベースに、本キーボード専用のシールドモジュール [ar-kr/tws_lp_shield_module](https://github.com/ar-kr/tws_lp_shield_module) として開発されています。ビルド済みのファームウェアは同リポジトリの **Releases** から配布されるので、通常はそれをダウンロードして書き込むだけでキーボードが動作します。

押さえておきたい仕様は次のとおりです。

- **セントラルは右手側**: PCやスマートフォンと直接BLE接続するのは右手側です。左手側は右手側と無線で繋がります。多くの分割キーボードと逆なので注意してください。
- **設定変更ツールはZMK Studio**: [ZMK Studio](https://zmk.studio/) でキーマップを変更します。USB接続＋WebHID対応ブラウザ（Chrome/Edge）が必要です。
- **macOSのJIS変換キー対応**: 拡張NKROレポートが有効化済みのため、`全/半`、`無変換`、`変換`、`かな` がmacOS / iPadOS でも動作します。
- **スリープ対応**: 30秒でアイドル状態、15分で完全スリープに入る省電力設定が入っています。

### 6.2 ファームウェアの入手

1. [ar-kr/tws_lp_shield_module のReleasesページ](https://github.com/ar-kr/tws_lp_shield_module/releases) を開きます。
2. 最新リリースの **Assets** から、添付されている `.uf2` ファイルをダウンロードします。リリースに含まれるのは次の3つです。

| ファイル名 | 用途 |
|---|---|
| `seeeduino_xiao_ble-tws_lp_jis60_right-rgbled_adapter-zmk.uf2` | 右手側（セントラル） |
| `seeeduino_xiao_ble-tws_lp_jis60_left-rgbled_adapter-zmk.uf2` | 左手側 |
| `seeeduino_xiao_ble-settings_reset-zmk.uf2` | 設定リセット用 |

> 自分でカスタマイズしたファームウェアを使いたい場合は、リポジトリをフォークしてGitHub Actionsでビルドすれば独自のuf2を作れます。手順は本章末尾の「6.7 ファームウェアのカスタマイズ」を参照してください。

### 6.3 書き込み手順

XIAO nRF52840 は **uf2ブートローダー** を搭載しており、PCに「USBドライブ」として認識させてファイルをドラッグ&ドロップするだけで書き込めます。専用ツールは不要です。

書き込みは **右手側 → 左手側の順** を推奨します。セントラル側を先に確定させたほうが、後で左右ペアリングがスムーズです。

#### 各キーボードの書き込み手順

1. キーボードをUSBケーブルでPCに接続します。
2. XIAOのリセットボタン（基板上の小さなボタン）を **2回素早く** 押します。
3. PCに `XIAO-SENSE` などの名前でUSBドライブが出現すれば、ブートローダーモードに入った状態です。
4. **設定リセット用 uf2 を先にドラッグ&ドロップ** します。書き込みが完了すると自動で再起動します。
5. もう一度リセットボタンを2回押してブートローダーに戻し、**本番用の uf2**（右なら `..._right_...uf2`、左なら `..._left_...uf2`）をドラッグ&ドロップします。
6. 書き込み後、自動で再起動して通常モードで起動します。

> リセット用uf2を先に入れる手順は、過去のペアリング情報や設定が残っていた場合のトラブルを防ぐためです。新品のXIAOであっても、最初の1回は実施することをおすすめします。

両方の書き込みが終わったら、いよいよ起動・ペアリングです。

### 6.4 起動とペアリング

#### 初回ペアリング

1. 右手側・左手側の両方の電源スイッチをONにします。
2. PCの Bluetooth 設定画面を開きます。
3. デバイス一覧に **`tws_lp_jis60 Right`** という名前が現れたら選択してペアリングします。左手側（`tws_lp_jis60 Left`）はPCからは見えません（PCには接続せず、右手側とのみ通信します）。
4. 右手側がPCにペアリングされると、数秒〜十数秒で左右間の接続も自動的に確立します。
5. 右手・左手それぞれの任意のキーを押して、PC側で文字入力されるか確認します。

#### BLEプロファイル切替（最大5台）

ZMKは最大5台のホストデバイスとペアリングを保持できます。

- **Lower（`Fn` ホールド）+ 1〜5キー** で各プロファイルに切り替えられます。
- **Lower + `Z`（`CLR`）** で現在選択中のプロファイルをクリア。
- **Lower + `X`（`CLRA`）** で全プロファイルをクリア。

別のPCやスマートフォンに接続したい場合は、未使用のプロファイル番号に切り替えてから新しいデバイスでペアリングしてください。

#### 左右が繋がらないときの復旧

1. 両側の電源をOFFにします。
2. 両側ともリセット用 uf2 を書き込みます（6.3節の手順を再実行）。
3. 両側に本番用 uf2 を書き戻します。
4. 右手側を先にONにし、PCでペアリング。
5. その後、左手側をONにします。

### 6.5 デフォルトキーマップ

デフォルトキーマップは2レイヤー構成です。

#### Default Layer（通常入力）

```
+-----+----+----+----+----+----+----+    +----+----+----+----+----+----+----+----+
| ESC |  1 |  2 |  3 |  4 |  5 |  6 |    |  7 |  8 |  9 |  0 |  - |  ^ |  \ |BSPC|
+-----+----+----+----+----+----+----+    +----+----+----+----+----+----+----+----+
| TAB |  Q |  W |  E |  R |  T |             |  Y |  U |  I |  O |  P |  @ |  [ |ENT |
+-----+----+----+----+----+----+             +----+----+----+----+----+----+----+----+
|CTRL |  A |  S |  D |  F |  G |             |  H |  J |  K |  L |  ; |  : |  ] |
+-----+----+----+----+----+----+             +----+----+----+----+----+----+----+
|SHFT |  Z |  X |  C |  V |  B |             |  N |  M |  , |  . |  / |  \ | ↑  |SHFT|
+-----+----+----+----+----+----+             +----+----+----+----+----+----+----+----+
|CTRL | Fn |GUI |ALT |EISU|SPC |             |SPC |KANA|GUI |ALT | Fn | ←  | ↓  | →  |
+-----+----+----+----+----+----+             +----+----+----+----+----+----+----+----+
```

ポイント:
- 中段左端は `CTRL`（HHKB配置に合わせた配置）
- `EISU` = `LANG2`（英数）、`KANA` = `LANG1`（かな）。macOSでは入力ソース切替に直接対応。
- 右上の `\` は `¥`（JIS_YEN）、`^` も同じくJISキー定義のマクロ。

#### Lower Layer（`Fn` ホールド中）

```
+-----+----+----+----+----+----+----+    +----+----+----+----+----+----+----+----+
|     | F1 | F2 | F3 | F4 | F5 | F6 |    | F7 | F8 | F9 |F10 |F11 |F12 |    |    |
+-----+----+----+----+----+----+----+    +----+----+----+----+----+----+----+----+
|     |    |BT1 |BT2 |BT3 |BT4 |BT5 |         |    |    |    |    |    |    |    |    |
+-----+----+----+----+----+----+              +----+----+----+----+----+----+----+----+
|     |CLR |CLRA|    |    |    |              |    |    |    |    |    |    |    |
+-----+----+----+----+----+----+              +----+----+----+----+----+----+----+
|     |    |    |    |CON |BAT |              |CON |BAT |    |    |    |    |    |    |
+-----+----+----+----+----+----+              +----+----+----+----+----+----+----+----+
|     |    |    |    |    |    |              |    |    |    |    |    |    |    |    |
+-----+----+----+----+----+----+              +----+----+----+----+----+----+----+----+
```

| 表記 | 機能 |
|---|---|
| `F1`〜`F12` | ファンクションキー |
| `BT1`〜`BT5` | BLEプロファイル 0〜4 を選択 |
| `CLR` | 現在のBLEプロファイルをクリア |
| `CLRA` | 全BLEプロファイルをクリア |
| `CON` | LEDで現在の接続状態を表示 |
| `BAT` | LEDで電池残量を表示 |

### 6.6 キーマップの変更（ZMK Studio）

キーマップをGUIで編集できる **ZMK Studio** が右手側で有効化されています。再ビルド・再書き込み不要で、ブラウザから直接キー割り当てを変更できます。

#### 接続手順

1. ChromeまたはEdge（WebHID対応ブラウザ）で [https://zmk.studio/](https://zmk.studio/) を開きます。
2. 右手側を USBケーブルでPCに接続します（ZMK StudioはUSB接続が必要です）。
3. ブラウザの「接続」ボタンをクリックし、デバイス選択ダイアログで `tws_lp_jis60 Right` を選びます。
4. 現在のキーマップが画面上に表示されます。

#### 編集と書き込み

- キーをクリックして割り当てを変更します。
- 編集が終わったら **Save** で本体に書き込み、即座に反映されます。
- 設定はXIAO内のフラッシュに保存され、電源を切っても保持されます。

> ZMK Studioで変更したキーマップは、本体のフラッシュ内に保持される「上書き設定」です。リセット用uf2を書き込むと初期キーマップに戻ります。

### 6.7 ファームウェアのカスタマイズ（任意）

ZMK Studio で実現できない変更（マクロ追加、新規レイヤー、コンボ、タップダンスなど）を行いたい場合は、シールドモジュールをフォークして自前でビルドする必要があります。

おおまかな流れは次のとおりです。

1. [ar-kr/tws_lp_shield_module](https://github.com/ar-kr/tws_lp_shield_module) を自分のアカウントへフォーク。

3. GitHub Actions の `Build ZMK firmware` ワークフローを実行。
4. ビルド成果物（Artifacts）から自作uf2を取得して、6.3節の手順で書き込み。

ZMKのキーマップ構文・カスタマイズの詳細は [ZMK公式ドキュメント](https://zmk.dev/docs) を参照してください。本ガイドではこれ以上踏み込みません。

## 7. 動作確認

ファームウェア書き込み・ペアリングが完了したら、すべての機能が正しく動くか確認します。問題があれば8章のトラブルシューティングへ進みます。

### 7.1 全キー導通テスト

すべてのキーが正しく入力されるかをテストします。

1. PCのブラウザで [keyboardchecker.com](https://keyboardchecker.com/) を開きます。
2. キーボードのすべてのキーを **1つずつ** 押していきます。
3. 押したキーが画面上で反応すればOK。

#### 反応しないキーがあった場合

そのキーに対応する箇所のはんだ付けに不良があります。Charlieplex方式の特性上、**1つのダイオード／ソケット不良が複数キーに影響する** 場合があるので、症状から該当箇所を推測する必要があります。

- **1キーだけ** 反応しない → そのキーのソケット or ダイオードのはんだ不良
- **複数キーが同時に** 反応しない → 共通のGPIO線につながる部分（ダイオードの片側、配線）の不良

詳しくは 8章「トラブルシューティング」を参照してください。

### 7.2 BLEプロファイル切替の確認

最大5台のホストとのペアリングを保持できる機能が動作するか確認します。

1. `Fn + 2`（Lower + 2）でプロファイル1に切り替え
2. 別のデバイス（スマホ等）で右手側を新規ペアリング
3. キー入力が新しいデバイスに送られることを確認
4. `Fn + 1` で元のデバイスへ戻れることを確認

#### 不要なペアリングを削除

特定プロファイルをクリアしたい場合は、対象プロファイルに切り替えてから `Fn + Z`（`CLR`）。
全クリアは `Fn + X`（`CLRA`）。

### 7.3 macOS / iOS / iPadOS でのJIS変換キー動作

本ファームは拡張NKROレポート対応のため、次のキーが Apple 製OSでも動作します。

| キー | 機能 |
|---|---|
| `EISU` キー（左下段） | 英数入力モード切替 |
| `KANA` キー（右下段） | かな入力モード切替 |
| `全/半` キー | 半角/全角切替（macOSでは英数キーと同等動作） |
| `無変換` / `変換` | IMEの変換操作（macOSではIMEによって挙動が異なる） |

iPad/iPhone と組み合わせる場合は、Bluetooth設定で `Japanese - Romaji` などの配列に設定するとより自然に動作します。

> **TBD**: 各OS（Windows / macOS / iPadOS / Linux / Android）での動作確認結果と注意点

### 7.4 LED状態表示の確認

XIAO nRF52840 のオンボードRGB LEDが、[zmk-rgbled-widget](https://github.com/caksoylar/zmk-rgbled-widget) により状態に応じて次のように点灯します。

#### 電池残量（起動時および `Fn + B` 操作時）

| 残量 | LED色 |
|---|---|
| 80%以上 | 🟢 緑 |
| 20〜80% | 🟡 黄 |
| 20%未満 | 🔴 赤 |
| 5%未満（危険） | 🔴 赤（残量変化のたびに点滅） |

#### BLE接続状態（`Fn + V` 操作時）

| 状態 | LED色 |
|---|---|
| 接続済み | 🔵 青 |
| アドバタイジング（ペアリング待ち） | 🟡 黄 |
| 切断 | 🔴 赤 |
| USB優先接続中 | シアン（水色） |

#### レイヤー表示

アクティブレイヤーの番号に応じて、シアン色でN回点滅します（0始まり。Default = 0回、Lower = 1回）。

### 7.5 電池駆動時間の確認

組み立て後、フル充電状態から徐々に電池が減っていく挙動を1〜2週間ほど観察し、自分の使用環境での電池駆動時間を把握しておきます。

- ZMKは適切にスリープすれば月単位で持続することが多い
- スリープしない設定や、左右の片側だけが起きっぱなしになる症状があると数日で切れる

## 8. トラブルシューティング

組立中・組立後によくある症状と対処方法をまとめます。

### 8.1 はんだ付け関連

#### 特定のキーが反応しない

最も多いトラブルです。次の順で原因を切り分けます。

1. **ホットスワップソケットの目視確認**: 該当キーのソケットが浮いていないか、はんだ付けに艶があるか
2. **ダイオードの極性確認**: 線（カソードバンド）の向きが他のダイオードと揃っているか
3. **ダイオードのはんだ付け確認**: 該当行・列の他のキーも反応しないかを確認（Charlieplexのため複数キーが連動する）
4. **スイッチ本体の不良**: 別のスイッチに差し替えてみる
5. **キーキャップが正しくセットされていない**: 押し下げきれていないケース

#### 複数のキーが同時に反応しない

Charlieplex方式では、特定のGPIOピンや配線で問題が起きると複数キーが連動して反応しなくなります。

- 反応しないキーをマトリクス上にマーキングしてみる
- 同じ列／行のキーが全滅していれば、その線につながる部品（ダイオードの片側、XIAOのピン周辺）を疑う
- XIAOのはんだ付けに不良がないか確認

#### キーが押しっぱなし扱いになる

- 該当キーのソケット内部にショートを引き起こす異物（はんだクズ・スイッチピンの曲がり）がないか確認
- 念のため、隣接キーのはんだブリッジも確認

### 8.2 ファームウェア・BLE関連

#### PCに認識されない（ブートローダーモード）

XIAO がブートローダーモードに入らないと、uf2ファイルを書き込めません。

- リセットボタンを **2回素早く** 押せているか確認（1回押すだけだと通常リセット）
- USBケーブルが **データ通信対応** か確認（充電専用ケーブルでは認識されない）
- 別のUSBポートやケーブルで試す
- それでもダメな場合、XIAO本体の故障の可能性

#### 左右が繋がらない

セントラル（右手側）と末梢（左手側）の接続が確立しない場合の手順:

1. 両方の電源をOFF
2. 両方とも **設定リセット用uf2** を書き込み（6.3節参照）
3. 両方とも **本番用uf2** を書き直し
4. 右手側を先に電源ON、PCでペアリング
5. 右手側がPCに接続できたのを確認してから左手側を電源ON

#### PCに認識されない（BLE）

- 右手側の電池残量が十分か確認
- PCのBluetoothがONになっているか、機内モードでないか確認
- 過去のペアリング情報が残っていることがあるので、PC側からデバイスを一度削除して再ペアリング
- 右手側でプロファイル切替（`Fn + 1`）を試す

#### 入力が一部しかPCに送られない

- 左手側だけが反応しない → 左右の接続不良。8.2「左右が繋がらない」を参照
- 右手側だけが反応しない → 右手側はセントラルなので、通常は右が動かなければ左も動かない。プロファイル切替や再ペアリングを試す

### 8.3 電源・電池関連

#### 電池がすぐ切れる

ZMKは適切に設定すれば月単位で持つはずです。短時間で切れる場合は次を疑います。

- **スリープ設定が無効**: `tws_lp_jis60_left.conf` / `tws_lp_jis60_right.conf` で `CONFIG_ZMK_SLEEP=y` および `CONFIG_ZMK_IDLE_SLEEP_TIMEOUT` が設定されているか
- **左右どちらかが常時起き続ける**: 何らかの理由でBLE接続が切れずスリープに入れない状態
- **バッテリー自体の劣化**: 古いバッテリーは容量低下、充電サイクル超過で持ちが悪化
- **常時何かのキーが押されている**: 8.1「キーが押しっぱなし扱いになる」を参照

#### 充電できない

- USBケーブルがデータ通信対応かを確認（充電できないケースは稀だが、極端に古い/粗悪なケーブルでは発生）
- XIAOの充電LED（橙色）が点くか確認
- バッテリーが寿命を迎えていないか

### 8.4 入力品質関連

#### チャタリング（1回の打鍵が複数回入力される）

- ZMKのデバウンス設定を見直す（`CONFIG_ZMK_KSCAN_DEBOUNCE_PRESS_MS` / `CONFIG_ZMK_KSCAN_DEBOUNCE_RELEASE_MS`）
- スイッチ自体の不良の可能性。ホットスワップなので差し替えて検証

#### キー入力のもたつき・遅延

- 電池残量が極端に低い場合、BLE接続が不安定になることがある
- 他のBluetooth機器が近くで干渉している場合は離してみる
- 2.4GHz Wi-Fiルーターの近くではノイズで通信品質が落ちる

### 8.5 それでも解決しない場合

- [`ar-kr/tws_lp_jis60` のIssues](https://github.com/ar-kr/tws_lp_jis60/issues) で類似の症状が報告されていないか検索
- [`ar-kr/tws_lp_shield_module` のIssues](https://github.com/ar-kr/tws_lp_shield_module/issues) も合わせて確認
- 新規Issueとして、症状・試したこと・写真を添えて報告
